# Gitコマンド定着テスト

Gitは「読む」だけでは定着しにくいので、**状況を見て、制限時間内にコマンドを打つ**練習ページです。
最初はコマンドを見ながらでOK。慣れてきたら、ページを閉じてタイマーだけで挑戦します。

## 使い方

1. 下の「シチュエーション」を読む
2. タイマーを開始する
3. 自分のローカル検証用リポジトリでコマンドを打つ
4. 終わったら `git status` や `git log --oneline` で確認する
5. 結果を定着度に反映する

!!! warning "本番リポジトリで練習しない"
    `reset` や `rebase` など履歴を動かすコマンドは、必ず練習用リポジトリで試してください。

## 定着度の見方

| 定着度 | 状態 | 目安 |
|---|---|---|
| ★☆☆ | 見ればできる | ページやメモを見ながら成功できる |
| ★★☆ | だいたいできる | 迷うが、制限時間内に成功できる |
| ★★★ | 反射でできる | 状況を見た瞬間にコマンドが出る |

## よく使うコマンド一覧

| コマンド | よくある用途 | 目標タイム | 定着度 |
|---|---|---:|---|
| `git status` | 今の状態を見る | 10秒 | ★★★ |
| `git add` | 変更をステージングする | 20秒 | ★★★ |
| `git commit` | 変更を履歴として保存する | 30秒 | ★★★ |
| `git pull` | リモートの最新を取り込む | 30秒 | ★★☆ |
| `git push` | ローカルのコミットを共有する | 30秒 | ★★☆ |
| `git branch` | ブランチ確認・作成 | 30秒 | ★★☆ |
| `git switch` / `git checkout` | ブランチを切り替える | 30秒 | ★★☆ |
| `git stash` | 作業中の変更を一時退避する | 60秒 | ★★☆ |
| `git log --oneline` | 履歴を短く確認する | 20秒 | ★★☆ |
| `git diff` | 変更差分を見る | 30秒 | ★★☆ |
| `git restore` | 作業ツリーの変更を戻す | 45秒 | ★☆☆ |
| `git reset` | ステージングやコミットを戻す | 60秒 | ★☆☆ |
| `git rebase` | 履歴を整理する | 120秒 | ★☆☆ |

## シチュエーション別テスト

### 1. いま何が変わっているか確認したい

**状況**：作業後、どのファイルを変更したか忘れた。  
**制限時間**：10秒  
**答え**：

```bash
git status
```

**追加で確認するなら**：

```bash
git diff
```

### 2. 変更をコミットしたい

**状況**：1つの作業が終わったので、履歴として保存したい。  
**制限時間**：1分  
**答え**：

```bash
git status
git add 対象ファイル
git commit -m "変更内容を短く書く"
```

### 3. 作業中の変更を退避して、ビルド結果を見せたい

**状況**：まだコミットしたくない変更がある。でも、作業前の状態でビルドを通して画面を見せる必要がある。  
**制限時間**：1分  
**答え**：

```bash
git status
git stash push -m "作業中: ビルド確認前の退避"
# ここでビルドや確認を実行する
git stash pop
```

**確認ポイント**：

- `git stash list` で退避が残っているか確認できる
- `git stash pop` は適用後にstashを削除する
- 不安な時は `git stash apply` を使うとstashを残したまま戻せる

### 4. 最新のmainを取り込んでから作業したい

**状況**：作業前にリモートの最新状態へそろえたい。  
**制限時間**：1分  
**答え**：

```bash
git switch main
git pull origin main
git switch 作業ブランチ
```

### 5. 間違えてステージングしたファイルを戻したい

**状況**：`git add` したが、まだコミット対象にしたくないファイルが混ざった。  
**制限時間**：45秒  
**答え**：

```bash
git restore --staged 対象ファイル
```

### 6. 直前のコミットメッセージを直したい

**状況**：コミット内容は正しいが、メッセージだけ間違えた。  
**制限時間**：1分  
**答え**：

```bash
git commit --amend -m "正しいコミットメッセージ"
```

!!! warning
    すでに共有済みのコミットを書き換える時は、チームの運用ルールを確認してください。

## 記録テンプレート

練習した日をメモして、定着度を更新します。

| 日付 | シチュエーション | 成功/失敗 | 秒数 | メモ |
|---|---|---|---:|---|
| 例: 2026-06-22 | stashしてビルド確認 | 成功 | 48秒 | `pop` と `apply` の違いを復習 |
