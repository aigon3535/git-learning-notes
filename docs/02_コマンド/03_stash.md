# stash

`git stash` は、**まだコミットしたくない作業中の変更を一時退避する**コマンドです。
「今の作業をいったん横に置いて、別ブランチへ移動したい」「作業前の状態でビルド確認したい」といった時に使います。

## よく使う場面

- 作業途中だけど、急ぎの修正ブランチに切り替えたい
- 未コミットの変更を残したまま、作業前の状態で動作確認したい
- 変更をコミットするほどではないが、一時的に避けたい

## 基本コマンド

```bash
git stash push -m "退避理由を書く"
```

退避した内容を確認します。

```bash
git stash list
```

退避した内容を戻します。

```bash
git stash pop
```

`pop` は戻した後にstashから削除します。残したまま戻したい時は `apply` を使います。

```bash
git stash apply stash@{0}
```

## ファイルを指定して退避する

全部ではなく、一部のファイルだけ退避したい時に使います。

```bash
git stash push -m "一部だけ退避" path/to/file
```

## 定着テスト

### 作業中の変更を退避して、ビルド結果を見せる

**制限時間**：1分  
**状況**：未コミットの変更があるが、作業前の状態でビルド結果を見せたい。  

```bash
git status
git stash push -m "作業中: ビルド確認前の退避"
# ビルドや画面確認をする
git stash pop
```

## 注意点

- stashは「一時置き場」であり、長期保存には向きません
- 複数stashすると `stash@{0}` `stash@{1}` のように積まれます
- `pop` でコンフリクトした場合、stashが残ることがあります
- 大事な作業はstashに置きっぱなしにせず、コミットやブランチで管理します
